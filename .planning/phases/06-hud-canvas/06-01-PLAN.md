# 06-01: Build VR HUD Canvas

<objective>
Generate the World Space Canvas HUD parented to the Main Camera. Attach three TMP_Text elements matching the specified dimensions, scales, anchors, and active states. Properly link these references to the GameManager script.
</objective>

<tasks>

  <task>
    <description>Create HUD Canvas and Child Elements</description>
    <read_first>
      - .planning/phases/06-hud-canvas/06-01-CONTEXT.md
    </read_first>
    <action>
      Use an Editor script (`Assets/Editor/HUDBuilder.cs`) to procedurally:
      1. Create a `Canvas` named `HUD_Canvas`. Set RenderMode to World Space, RectTransform size to 1920x1080.
      2. Identify the `Main Camera`. Parent the Canvas to it and set Local Position (0, 0, 1.5) and Scale (0.0005, 0.0005, 0.0005).
      3. Create three `TextMeshProUGUI` children (`TimerText`, `ExtinguisherText`, `ResultText`).
      4. Assign defined RectTransform anchors, dimensions, font sizes, colors, alignment, and initial text.
      5. Set `TimerText` enabled and the other two disabled.
      6. Automatically find `GameManager` and assign these three references to their matching fields.
    </action>
    <acceptance_criteria>
      - Editor script successfully compiles and executes.
      - Canvas created and parented to Camera.
      - Elements positioned and styled correctly.
      - GameManager Inspector links populated.
    </acceptance_criteria>
  </task>

  <task>
    <description>Verify SelectionScreen Decoupling</description>
    <read_first>
      - .planning/phases/06-hud-canvas/06-01-CONTEXT.md
    </read_first>
    <action>
      Within the same Builder script, query for the Phase 2 `SelectionScreen` Canvas. Force its parent to `null` to detach from any Camera hierarchy and statically set its World position to `(0, 1.5, 3)`.
    </action>
    <acceptance_criteria>
      - `SelectionScreen` is proven independent and stationed at the requested coordinates.
    </acceptance_criteria>
  </task>

</tasks>

<verification>
### Automated Verification
None (scene manipulation script)

### Manual Verification
1. Run `VR Firefighter > Build HUD Canvas` from the Unity editor menu.
2. Ensure no compiler errors exist.
3. Validate GameManager inside the hierarchy correctly references the generated UI objects in its Inspector slots.
</verification>

<must_haves>
- Canvas parented to Main Camera to emulate XR HUD behavior natively.
- Use `TMP_Text` specifically so it binds accurately to GameManager's properties.
</must_haves>
