# Plan 02-03: Scene Wiring & State Verification

<objective>
Wire all Inspector references for ScenarioSelector and GameManager, then verify the complete Phase 2 setup: SelectionScreen visible at start, Kitchen/ServerRoom inactive, A button activates Kitchen, B button activates ServerRoom.
</objective>

<must_haves>
- ScenarioSelector Inspector references wired correctly
- GameManager object exists in scene with GameManager.cs
- Scene state correct at Play: SelectionScreen active, Kitchen/ServerRoom inactive
- A press activates Kitchen and hides SelectionScreen
- B press activates ServerRoom and hides SelectionScreen
</must_haves>

## Tasks

<task id="02-03-01">
<title>Create GameManager object and wire ScenarioSelector references</title>

<read_first>
- .planning/phases/02-scenario-selection-screen/02-CONTEXT.md (D-12, D-13, D-22 — wiring spec)
</read_first>

<action>
In the Unity scene:

1. Create empty GameObject named **"GameManager"** at position (0, 0, 0)
   - Attach `GameManager.cs` script
   - Attach `ScenarioSelector.cs` script

2. In Inspector, wire ScenarioSelector references:
   - **selectionScreen** → drag SelectionScreen Canvas object
   - **kitchenRoot** → drag Kitchen object
   - **serverRoomRoot** → drag ServerRoom object
   - **gameManager** → drag the GameManager component (self-reference on same object)

All four references must be assigned — none should be null.
</action>

<acceptance_criteria>
- GameManager GameObject exists in scene at root level
- GameManager has GameManager.cs component attached
- GameManager has ScenarioSelector.cs component attached
- ScenarioSelector.selectionScreen references SelectionScreen object
- ScenarioSelector.kitchenRoot references Kitchen object
- ScenarioSelector.serverRoomRoot references ServerRoom object
- ScenarioSelector.gameManager references GameManager component
- No null references in ScenarioSelector Inspector
</acceptance_criteria>
</task>

<task id="02-03-02">
<title>Verify scene state and button functionality</title>

<read_first>
- .planning/ROADMAP.md (Phase 2 success criteria)
- .planning/REQUIREMENTS.md (SEL-01 through SEL-06)
</read_first>

<action>
Verification checklist — enter Play mode and confirm:

**Initial State:**
1. SelectionScreen canvas is visible and centered in front of player
2. Both KitchenCard (orange) and ServerCard (blue) text are readable
3. Kitchen GameObject is inactive (not visible in scene)
4. ServerRoom GameObject is inactive (not visible in scene)

**A Button Test:**
5. Press A (buttonSouth) on Xbox controller
6. Confirm: SelectionScreen disappears
7. Confirm: Kitchen becomes active
8. Confirm: Console shows "StartGame called with scenario: Kitchen"

**B Button Test (restart Play mode):**
9. Press B (buttonEast) on Xbox controller
10. Confirm: SelectionScreen disappears
11. Confirm: ServerRoom becomes active
12. Confirm: Console shows "StartGame called with scenario: ServerRoom"

**Error Check:**
13. Unity Console shows zero errors
14. No warnings about missing references

If no Xbox controller is available, verify visually that:
- SelectionScreen renders at correct position
- Cards have correct colors and text
- Kitchen and ServerRoom are inactive in Hierarchy
- All ScenarioSelector references are wired (no null)
</action>

<acceptance_criteria>
- In Play mode: SelectionScreen canvas visible at game start, positioned at (0, 1.5, 3)
- In Play mode: Kitchen and ServerRoom objects are inactive at start
- With controller: A press activates Kitchen, deactivates SelectionScreen
- With controller: B press activates ServerRoom, deactivates SelectionScreen
- Console shows "StartGame called with scenario: Kitchen" or "ServerRoom" on selection
- Zero compile errors in Unity Console
- Zero null reference warnings
</acceptance_criteria>
</task>
