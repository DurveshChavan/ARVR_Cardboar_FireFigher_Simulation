# Plan: Replace Cardboard XR Plugin with ManualVRRig

## Task 1: Write ManualVRRig.cs

**Files:**
- `VR_Firefighter/Assets/Scripts/ManualVRRig.cs` [CREATE]

**Action:**
Create the complete ManualVRRig MonoBehaviour with:
1. Two Camera references (LeftEye, RightEye)
2. IPD field (default 0.064f)
3. Neck (Transform) and Head (Transform) references for the split rotation architecture
4. Stereo rect assignment on Start
5. Gyroscope init and coordinate conversion in Update
6. Gamepad right-joystick yaw/pitch on the Neck object
7. Using `Input.gyro` (legacy Input API — works on Android without Input System configuration)
8. Gamepad reading via `UnityEngine.InputSystem.Gamepad.current` (New Input System) with null guard
9. Sensitivity fields for joystick look speed

**Coordinate conversion math (critical):**
```
// Mobile gyro is right-handed, Unity is left-handed
// Raw gyro attitude: (x, y, z, w)
// Converted: new Quaternion(raw.x, raw.y, -raw.z, -raw.w)
// Then rotate -90 degrees on X to map "phone held upright" -> "looking forward"
// Final: Quaternion.Euler(-90, 0, 0) * converted
```
